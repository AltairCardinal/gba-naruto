# 续作开发工作区

## 目标

把当前仓库从“只有逆向记录”推进到“可以持续做续作开发”的状态。

这不等于已经完全拿到剧情脚本导入格式，而是先把下面四件事固定下来：

1. 项目元数据
2. 内容作者工作区
3. ROM 构建入口
4. 每次改动的持久记录方式

## 目录

- `sequel/project.json`
  项目级元数据，约束基础 ROM、输出路径和补丁清单。
- `sequel/patches/manifest.json`
  当前实际会写回 ROM 的补丁入口。
- `sequel/content/`
  续作内容源稿。即使当前还不能完整导入 ROM，也必须先把设计内容存放在这里。
- `tools/build_mod.py`
  统一构建入口。

## 规则

1. 任何有效写回改动都必须进入 `sequel/patches/manifest.json` 或被其引用。
2. 任何有效的剧情、关卡、单位设计都必须进入 `sequel/content/`。
3. 任何新逆向结论都必须更新 `notes/` 或 `docs/`。

## 当前状态

当前构建器已经支持：

- 校验基础 ROM 的 SHA-1
- 复制基础 ROM 到构建目录
- 应用显式字节补丁
- 输出构建报告 JSON

这让“可重复构建的 ROM 开发流程”先成立。后续只要定位到更高层格式，就继续往同一个工作区里接：

- 文本导入器
- 地图导入器
- 指针重定向器
- 空闲区分配器
